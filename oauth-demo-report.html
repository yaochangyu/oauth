演練範圍決策

為何選 MVC 而非 SPA？
任務要求「先針對 SPA 頁面，若沒有 SPA 頁面，再選 ASP.NET Core MVC」。
實際檢查 src/Clients/ 目錄，只存在兩個 .NET 專案： OAuth.Client.Mvc/（MVC）與 OAuth.Client.WebAPI/（Web API）。 無任何 package.json、vite.config.*、.html 入口或 Vue/React 原始碼。 README 也僅描述 MVC 與 WebAPI 兩個示範客戶端。因此本次演練以 MVC Cookie SSO 為主要瀏覽器流程。
路徑存在說明
src/Clients/OAuth.Client.Mvc/✅ASP.NET Core 10 MVC，port 5101，Cookie SSO
src/Clients/OAuth.Client.WebAPI/✅ASP.NET Core 10 Web API，port 5102，Bearer Token
src/Clients/*/package.json❌ 不存在無 SPA 應用程式
src/Clients/*/vite.config.*❌ 不存在無 Vite/Vue/React 專案
為何使用 Playwright 而非 Webwright？
任務指示「優先使用 Webwright，再來才是 Playwright」。Webwright 技能已載入，但其執行依賴 webwright Python package 及其內建工具（webwright.tools.self_reflection 等）。 本環境中 import webwright 不可用（Webwright 是技能文件，非可安裝 package）。 故降階使用 Playwright Python API，並遵循 Webwright 的 workspace contract（final_runs/run_N/screenshots/ 命名規範、plan.md、action log）。 截圖證據等價，執行邏輯相同。

spa-client PKCE 協定驗證

⚠️ 注意：本節驗證 spa-client 的 PKCE 協定正確性（授權碼取得 + Token 換發）， 不代表 SPA UI 演練（因目前 src/Clients/ 沒有 SPA 應用程式）。 待使用者閱讀本報告後，可加入實際 Vue/React 前端再進行完整 SPA UI 演練。

spa-client 是 Public Client（不需 client_secret），使用 PKCE 代替 secret 保護授權碼。此次透過 Playwright 取得 authorization code 後，以 curl 換取 token。

步驟 1 — 產生 PKCE 參數
import hashlib, base64, os verifier = base64.urlsafe_b64encode(os.urandom(32)).rstrip(b'=').decode() challenge = base64.urlsafe_b64encode( hashlib.sha256(verifier.encode()).digest() ).rstrip(b'=').decode()
步驟 2 — 授權 Endpoint（Playwright 驅動）
GET https://localhost:7098/connect/authorize ?client_id=spa-client &response_type=code &scope=openid+profile+email+offline_access+api &code_challenge={challenge} &code_challenge_method=S256 &redirect_uri=https://localhost:5173/callback &response_mode=query
步驟 3 — Token Exchange（curl）
curl -sk -X POST https://localhost:7098/connect/token \ -H "Content-Type: application/x-www-form-urlencoded" \ -d "grant_type=authorization_code &code={authorization_code} &redirect_uri=https://localhost:5173/callback &client_id=spa-client &code_verifier={verifier}"
回應（HTTP 200）
{ "access_token": "eyJhbGci... (JWE, 15 分鐘有效)", "token_type": "Bearer", "expires_in": 899, "id_token": "eyJhbGci... (JWT, 含 sub/email/name)", "refresh_token": "eyJhbGci... (JWE, 30 天有效)", "scope": "openid profile email offline_access api" }
驗證項目結果
Authorization Code 取得✅ 成功（Playwright 攔截 callback URL）
PKCE code_verifier 驗證✅ 通過
Access Token（JWE）✅ 回傳
ID Token（JWT）✅ 含 sub / email / name
Refresh Token✅ offline_access scope 生效
Token 有效期✅ AT=899s，Scope 正確
SPA UI 截圖⚠️ 無（目前無 SPA 應用程式）

Client Credentials 協定驗證

服務對服務（M2M）認證，使用 webapi-client，不需要使用者介入。直接以 client_id + client_secret 換取 Access Token。

請求
curl -sk -X POST https://localhost:7098/connect/token \ -H "Content-Type: application/x-www-form-urlencoded" \ -d "grant_type=client_credentials &client_id=webapi-client &client_secret=webapi-client-secret &scope=api"
回應（HTTP 200）
{ "access_token": "eyJhbGci... (JWE 格式)", "token_type": "Bearer", "expires_in": 899 }
驗證項目結果
HTTP 狀態碼✅ 200 OK
Access Token（JWE）✅ 回傳
expires_in✅ 899 秒（15 分鐘）
Refresh Token不包含（Client Credentials flow 規格如此）

Refresh Token Rotation

每次使用 Refresh Token 換發 Access Token 時，同時回傳新的 Refresh Token，舊的立即失效，防止 RT 被竊後長期濫用。

換發請求
curl -sk -X POST https://localhost:7098/connect/token \ -H "Content-Type: application/x-www-form-urlencoded" \ -d "grant_type=refresh_token &refresh_token={old_refresh_token} &client_id=spa-client"
回應（HTTP 200）
{ "access_token": "eyJhbGci... (新 AT)", "token_type": "Bearer", "expires_in": 899, "refresh_token": "eyJhbGci... (新 RT，舊值已失效)", "scope": "openid profile email offline_access api" }
驗證項目結果
新 Access Token✅ has_access_token=True
新 Refresh Token（輪替）✅ new_rt_differs=True（與舊值不同）
Scope 保留✅ openid profile email offline_access api
AT 有效期✅ 899 秒（15 分鐘）
RT 有效期✅ 30 天
